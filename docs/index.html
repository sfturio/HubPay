Introduction

HubPay Overview

HubPay e um mini gateway de pagamentos criado para estudar arquitetura backend e padroes comuns em APIs de pagamentos. O projeto simula como sistemas de pagamento organizam autenticacao, criacao de pagamentos e mudancas de estado. O foco e demonstrar boas praticas de backend de forma clara e objetiva.

speed

High Performance

O projeto demonstra Minimal APIs e design orientado a performance para respostas rapidas e consistentes.

security

Secure by Design

Autenticacao por API key e requisicoes idempotentes para reforcar seguranca e confiabilidade.

System Architecture

HubPay utiliza uma arquitetura em camadas para separar responsabilidades e manter o dominio isolado de detalhes tecnicos. Essa estrutura facilita evolucao do sistema, testes e manutencao do codigo.

Payment Lifecycle

STEP 01

Pending

Pending e o estado inicial de um pagamento logo apos sua criacao. O pagamento foi registrado, mas ainda nao foi autorizado.

STEP 02

Authorized

Authorized indica que o pagamento foi aprovado e pode ser capturado. Esse estado representa uma autorizacao valida antes da conclusao do pagamento.

STEP 03

Paid

Paid representa a conclusao do pagamento. Nesse estado o valor foi efetivamente capturado e o pagamento e considerado finalizado.

Authentication

HubPay utiliza autenticacao baseada em API key para identificar o merchant que realiza a requisicao. Cada requisicao deve incluir a chave no header x-api-key para acessar os endpoints da API.

Request Header

x-api-key: sk_test_xxx

Idempotency

A criacao de pagamentos suporta idempotencia para evitar duplicacao de operacoes. Clientes podem enviar um Idempotency-Key para garantir que a mesma requisicao nao gere multiplos pagamentos.

lightbulb

Quando uma requisicao com a mesma chave e recebida novamente, o sistema retorna a resposta original em vez de criar um novo pagamento.

Core API Endpoints

Method Endpoint Description
POST /payments Cria um novo pagamento associado a um merchant e inicia o fluxo de processamento.
GET /payments/{id} Retorna os detalhes de um pagamento especifico usando seu identificador.
GET /payments?status=paid Lista pagamentos do merchant filtrando por status.
GET /payments?customerId={uuid} Lista pagamentos do merchant filtrando por cliente.
POST /payments/{id}/authorize Autoriza um pagamento previamente criado, movendo seu estado de Pending para Authorized.
POST /payments/{id}/pay Marca um pagamento autorizado como pago.
POST /payments/{id}/fail Marca um pagamento pendente/autorizado como falho.
POST /payments/{id}/refund Reembolsa um pagamento que ja foi concluido.

Webhooks

HubPay gera eventos quando o estado de um pagamento muda. Esses eventos podem ser enviados para endpoints de webhook registrados pelos merchants para permitir integracao com sistemas externos.
